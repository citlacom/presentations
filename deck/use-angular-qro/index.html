Organización del código

AngularJS te obliga a crear controladores específicos para cada parte de tu interfaz y un elemento HTML del DOM queda asociado.

En mi proyecto los principales controladores son:

  • Main - Controla inicialización de variables de configuración
  • SearchBox - Controla la caja de búsqueda
  • FacetGroup - Controla la inicialización de los facets
  • FacetField - Controla las opciones de cada facets (activación/desactivación)
  • Results - Controla la visualización general de los resultados y paginación
  • Result - Controla cada elemento individual de los resultados y activación de vista de detalle
  • DetailPage - Maneja la visualización de detalle de un resultado

Plantillas fáciles de escribir

AngularJS usa sintaxis HTML con opciones avanzadas muy fáciles de aprender como:

  • Valores de reemplazo - {{nombre_variable}} o {{objeto.propiedad}}
  • Atributo ng-repeat="tag in topic.tags" para iterar un array
  • Atributo ng-show="documents.length > 0" para condicionar visualización de elemento
  • Atributo ng-hide="documents.length <= 0" para condicionar ocultación de elemento
  • Atributo ng-click="handleSetPage(page.number)" para ejecutar función en el evento click

Ejemplo de Plantilla

  <!--Results summary-->
  <div class="search__messages" ng-show="documents.length > 0" ng-cloak>
    <div class="search__no-results" ng-hide="documents.length">{{ LABELS.NO_RESULTS }}</div>
    <div class="messages message--error alert alert-warning" ng-show="error">{{ error }}</div>
    <div class="messages message--info alert alert-info" ng-show="message">{{ message }}</div>
  </div>

  <!--Results display-->
  <div class="search__results-list" ng-repeat="document in documents">
      <resultdefault document="document"></resultdefault>
  </div>

  <!--Pagination-->
  <ul class="pagination">
    <li class="pager__item" ng-repeat="page in pages">
      <a ng-class="{iscurrent:page.isCurrent}" ng-class="{pager__item-current:page.isCurrent}"
        ng-click="handleSetPage(page.number)">{{ page.name }}</a>
    </li>
  </ul>

Preservar estado de la aplicación al recargar o compartir

AngularJS permite modificar el valor hash de la URL y utilizar esos valores para inicializar el estado de la aplicación:

  • $location.path()
  • $location.hash()
  • $location.url()
  • $location.search('foo');
  • $scope.$on('$locationChangeStart', function(event, next, current) {});

Fichero de configuración para adaptar mi aplicación fácilmente

La estructura modular de AngularJS y la posibilidad de declarar constantes que se inician desde una estructura JSON.

        var config_data = {
          'CONFIG': {
            'FACETS_ENABLE': true,
            // Indicates the minimum counts for facet fields should be included in the response
            'FACETS_MINCOUNT': 1,
            // Indicates the maximum number of constraint counts that should be returned for the facet fields
            'FACETS_HARD_LIMIT': 20,
          }
        }

        var module = angular.module('SolrSearchApp.config', []);
        angular.forEach(config_data, function(key, value) {
          module.constant(value, key);
        });

Sincronización de valores de variables en elementos de formulario

Cada controlador de AngularJS tiene una variable $scope cuyas propiedades recibes en las plantillas y con la que puedes asociar alguna a un elemento de fomulario.

  • <input type="text" class="form-control" id="query" type="text" ng-model="userQuery" />
  • $scope.userQuery se inicializa en el controlador con un valor que se alimenta de la URL
  • Si ese valor en la URL es alterado el elemento input lo reflejará de inmediato
  • No es necesario seleccionar el inptu y definir el valor como en jQuery: $('#id').val(myvalue);
